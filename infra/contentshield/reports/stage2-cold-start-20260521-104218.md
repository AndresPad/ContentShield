## Comparison summary

All on NC24-A100 (1×A100 80GB), cpu=24 vCPU / mem=220 GiB per replica.

| Variant | scale→replica | replica→health | total | first classify | warm p50 |
|---|---|---|---|---|---|
| baked-local (warm node) | 94s | 16s | **110s (1m50s)** | 2.31 s | **133.4 ms** |
| baked (cold node) | 732s | 622s | **1354s (22m34s)** | 2.28 s | 132.3 ms |
| cache-disabled (cold node) | 267s | 518s | **785s (13m05s)** | 2.28 s | 133.5 ms |

Cold-node-to-cold-node comparison (apples to apples — fresh GPU node, no
image cache, no model cache):

| Variant | image pull | model load | TOTAL cold start |
|---|---|---|---|
| baked (HF cache baked) | 12m12s | 10m22s | **22m34s** |
| cache-disabled (no bake) | 4m27s | 8m38s | **13m05s** |
| baked-local (offline) | (vendor: 12m13s) | (vendor: 3m02s) | **vendor 21m15s, real 15m+ expected** |

Apples-to-apples cold-node baked-local was not captured this run because we
got a warm node. Vendor matrix says ~21m15s e2e for fresh-node baked-local —
but the model-load delta is the killer:

- **baked-local model load: ~3m** (vendor) — purely local SSD read, offline.
- **baked model load: ~10m22s** (this run) — touches HF for cache validation.
- **cache-disabled model load: ~8m38s** (this run) — full HF download.

## Steady-state (warm-node) behaviour

For a scale-out event when the node *already has the image cached* (the
common production case after the first cold start):

- **baked-local: 110s to /health** (16s of that is the vLLM model load).
- baked & cache-disabled: model download/cache lookup still happens, so the
  warm-node case is dominated by HF connectivity, not local SSD.

## Recommendation

**baked-local for production.** It dominates everywhere except for the
58 GB image pull, which is one-time-per-node and fully solvable with
ACR artifact streaming (already enabled — convertPushedImages: true) and
a small minReplicas=1 floor on the critical path variant.

cache-disabled is competitive only when you cannot bake (vendor refuses
or weights aren't redistributable). The HF download has the same ~9 min
cost whether the image is small or large.

baked is strictly dominated: it pulls 58 GB AND still touches HF.

## Warm classify latency (apples-to-apples)

All three variants produced essentially identical warm p50/p95: ~133/134 ms.
This is the steady-state A100 single-request latency for label-only
guided-choice decoding on Gemma-4-31B-IT at MAX_MODEL_LEN=20000,
GPU_MEMORY_UTILIZATION=0.9, EXTRA_ENGINE_ARGS=--enable-prefix-caching.
